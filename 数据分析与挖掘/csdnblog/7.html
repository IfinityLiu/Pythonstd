Chrome 成为互联网看门人？！

浏览器纷争何时休？

作为全球第一大浏览器的 Chrome 于一夕之间成为了众矢之的。究其缘由，可以视为“树大招风”引来的祸端，亦可以看成“天下厂商苦于 Chrome 已久”。


开源项目的争论


近日，彭博社发表了一篇名为《Google’s Chrome Becomes Web ‘Gatekeeper’ and Rivals Complain》的报道，揭露了当前浏览器领域，Chrome 所处的现状以及 Google 的竞争对手们发出的无奈抱怨。

众所周知，以技术为驱动的科技巨头 Google 为开源界做出了重大的贡献。无论是全球第一大操作系统 Android 的 AOSP 项目，还是占据浏览器市场极高份额 Chrome 的 Chromium 项目，Google 都是其主导者之一。

因此对于不少开发者而言，如想要创建自己的浏览器，大多数会选择基于 Chromium 项目进行开发。

不过问题症结就出在了这里，对于开源项目的使用，开发者并没有太多的自由。据彭博社报道，Web 浏览器 Metastream 的开发者 Samuel Maddock 表示，在基于 Chromium 项目开发自己浏览器的过程中，他发现其主打“与朋友一起观看流媒体”的浏览器需要一种名为 Widevine 的数字版权服务产品来验证用户身份并防止视频盗版。

因此，他向 Google 发送了一份该服务的申请请求，在邮件中他在概述了该项目之后开始了漫长的等待。

万万没想到的是，在熬过了四个月和接连发送了 10 封电子邮件之后，他得到了 Google 一句冰冷的回复：对不起，你不能使用该软件。

Maddock 表示自己并没有做过任何非法行为。虽说使用 Google 的安全流媒体工具，Maddock 的项目也可以顺利进行，但 Widevine 产品是基于 Chromium 的浏览器唯一可用的 DRM（数字版权管理）。

Google 在拒绝了 Maddock 请求的同时，并没有回复说明具体的原因，这让 Maddock 难以接受，而在不久之后，Maddock 更是无奈放弃了浏览器研发，与此同时他也表示：“ Google 这样的看门人，完全可以决定哪些项目可以运作。如果没获得它的许可，那你就完了。”

事实上，有类似经历的并不只有 Maddock 一人，但在浏览器市场中，能对用户访问网络方式有如此大影响的厂商却并不多，而 Google 就是为数不多的厂商之一。


浏览器市场格局：Chrome 第一，Safari、Firefox 其次


放眼当前浏览器市场，根据美国知名网站通讯流量监测机构 StatCounter 数据统计，Google 的 Chrome 可谓是占据了全球浏览器市场的半壁江山，占比高达 62.8%，其次苹果的 Safari 占比 15.83%，Mozilla 的 Firefox 排名第三，占 4.86% 的市场份额，而彼时浏览器界的一哥 IE，早已不在 Top 5 之列。

从下面的线形图中，更可以直观地看到，自 2012 年 5 月之后，赶超 IE 的 Chrome 一飞冲天， 势不可挡。


“Chrome 成为互联网的看门人”


随着 Chrome 地位的逐步上升，无数开发者和厂商纷纷涌入其中。

除了上文所述的 Maddock 开发者之外，2013 年的 Opera、今年的 Edge 等浏览器厂商们都已切换到 Google 维护的开源项目 Chromium，如彭博社报道，这创造了一种滚雪球效应，越来越少的开发者会为非 Chrome/Chromium 浏览器优化。

在这种趋势之下，Google 的 Chrome 俨然成为诸多标准的引领者，甚至不少业界人士将其评价为「互联网的看门人」。

对此，前 Mozilla CTO Andreas Gal 评价道，无论 Chrome 做什么都是标准。

回忆在 Mozilla 公司工作的七年间，Gal 表示，某天，Google 的 Docs、Gmail 突然无法在 Firefox 上使用了。而 Firefox 的研发团队花费了两个月的时间去修复这些问题，但是在此期间，用户对于 Firefox 的直观印象就已经变差了，大家理所应当地认为是“Firefox 出现了 Bug”。

与此同时，Gal 还表示“今天，特别是在桌面领域，Google 绝对是一个垄断者。”

这种统治意味着 Google 设定了互联网的标准。在该愿景中，广告和用户数据收集是默认设置。

不过，Google Chrome 团队工程副总裁 Darin Fisher 为此进行了辩护，否认 Google 有意打压竞争对手。同时，Google 表示一直在寻找好的解决方案，如倡导更多公共的标准，以便所有的浏览器都可以遵循这些标准。

但是在一家独大的浏览器市场中，Google 的这些举措并没有迎来想象中的掌声，反而质疑声越来越盛。

其中 Mozilla 的联合创始人兼 Brave Software Inc. 现任 CEO Brendan Eich 表示，“Chrome 已经变成了间谍软件。”

Eich 称 2017 年 8 月，Netflix 突然停止使用 Brave 的浏览器。在一连串的电子邮件之后，其首席技术官 Brian Bondy 发现 Google 的更新改变了 Netflix 使用 Widevine 的方式。而 Brave 并没有被告知这些变化，所以当用户在线访问 Netflix 时，发现浏览器出现了问题。而修复问题花了两周多的时间。

此外，Opera 联合创始人 Jon von Tetzchner 也曾表示，于 2015 年发布的 Vivaldi 是一款面向隐私人群的流行浏览器，不过在运行 Docs 和 Gmail 等 Google 服务方面经常会遇到麻烦。一些在 Vivaldi 上登录 Google 产品的用户会收到提示，说他们的浏览器没有针对他们进行优化，或建议他们下载 Chrome。

“这非常明确地针对我们，”von Tetzchner 表示。


今天的 Chrome 是否会成为昨日的 IE？


基于以上的种种，Google 是否存在垄断行为，我们尚未可知。不过，近日美国司法部开始对 Alphabet 旗下子公司 Google 展开反垄断调查，而调查的重点就是与 Chrome 有一定关系的数字广告业务。

其实，对于开发者而言，Chrome 能在圈中如此受欢迎，并非是因为它背靠 Google 这颗大树，更多的还是其拥有简洁的 UI、良好的扩展性、便捷高效的多进程架构等优势。

正如 IE 的颓败，不是因为它没有微软为靠山，而是因为 IE 自身的不作为。

不知有所少人还能记得，IE 曾经也站在如今 Chrome 的位置，而在此 IE 之前，浏览器巨头 Netscape 同样也是。谈及一代浏览器 IE 的起伏， 彼时谁曾想到，2008 年才出现的 Chrome 仅用了四年的时间就取代了 IE 的位置。

深究 IE 失败的原因，盲目自信、无视市场生存法则是其走向衰败的主要原因。彼时凭借免费、迅捷的迭代速度，再加上全球第一大操作系统 Windows 的捆绑式加持，IE 轻松战胜了 Netscape 浏览器，然而打江山容易守江山难，IE 浏览器从 IE 6 开始变得盲目自信，中间一度有 5 年之久未曾有过迭代更新，除此之外，其不再参与 W3C 标准的制定，为此，忍无可忍的 Google YouTube 开发团队还曾于一顿简短的午餐期间密谋干掉 IE 6。后来虽然在 Firefox 的连番攻势下，微软幡然醒悟然为时已晚，且后来其更新的 IE 7、IE 8 因速度慢、占用资源大一直被用户所诟病。就此，IE 才走上了穷途末路。

如今 Chrome 虽然在功能上依旧保持最前沿的更新，但是在业务层面上是否会变成 IE 那般让人讨厌的模样？对此，你怎么看？

事实上，一家独大的局面，之于市场格局、用户、开发者都未必喜闻乐见。
